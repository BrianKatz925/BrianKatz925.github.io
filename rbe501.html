RBE501: Robot Dynamics

Summary

The final assignment of Robot Dynamics was the implementation of inverse dynamics control of a three degree of freedom robot arm in MATLAB. In this simulation, we defined transformation matrices, spatial inertia matrices, and applied inverse kinematics and quintic trajectory generation for robot motion. We included several scenarios, such as the robot falling under its own weight, changing theoretical link moment of inertias, and adding a payload to observe changes in joint torques.

To learn more about this project in detail, view my submission here.

Derivation and Simulation

To obtain the homogeneous transformation matrices, I broke up the problem into two components: finding the rotation matrix and the position vector. I used visual inspection to find the rotation matrix, as the coordinate systems for each frame were defined, and utilized the dot product between each axis to define the rotation between each frame. The position vectors were also obtainable from the robot diagram, as the link lengths were defined. Using a function called RpToTrans, I input the 3x3 rotation matrix and 1x3 position vector to form the 4x4 transformation matrix.

I then calculated the Spacial Inertia matrices, assuming for this portion that no rotatatioal inertia was on the links, letting them fall under their own weight in simulation. In the following solutions, I1,I2,and I3 are the rotational inertia matrices, m1,m2,and m3 are the weights of the links, and G1,G2, and G3 are the spacial inertia matrices.

Next, I calculated a set of quintic polynomial trajectories given a set of desired positions, velocities, and accelerations. The quinticpoly function below solves a system of linear equations based on fifth order polynomial equations and the state space variables: position velocity and acceleration. The solution of this system is a set of coefficients, which I input into the fifth order equations to define my position, velocity, and acceleration trajectories.

For the next problem, I was given a set of rotational inertia matrices, and placed them into spatial inertia matrices, following the previous method. Next, I calculated the joint torques required to move the robot along a spiral trajectory in the task space. To generate the list of target poses for the robot, I converted the list of task space coordinates into joint space coodinates using inverse kinematics. This was done using the 3x3 analytical Jacobian (which can only update the position of the end effector, not the orientation) and the Levenberg-Marquadt/Damped Least Squares algorithm to recursively update the position until the pose error is minimal.

Next, I generated a series of quintic trajectories from point to point, following the method described previously. To traverse these trajectories, I generated a series of wrenches using inverse dynamics, creating a matrix of joint torques to be applied to form a spiral. To simulate the robot moving from position to position, I used normal dynamics with the solved joint torque wrenches and plotted the robot at the solved joint positions.
